### 병사 관리 문제 
- 개요
  - 병사들을 관리하는 프로그램
  - 병사 각각 고유 번호, 소속팀(1~5), 평판 점수(1~5) 가짐
  - 기능(5): 고용, 해고, 평판 점수 변경, 특정 팀 병사 평판 점수 일괄 변경, 특정 팀 내 최고 평판 점수 병사 검색

- 특이점 
  - 함수별 호출 횟수
    - 네 함수는 100,000회 이하 호출하지만 하나는 100회 이하 호출


- 아이디어
  - 사람을 배열로 관리 시 찾고 관리하는 게 오래 걸림
  - 사람을 줄 세우는 게 아니라 같은 점수끼리 모으면 어떨까? (점수가 1~5뿐임)
    - 장점: 팀 점수 통틀어 바꿀 때 동일 점수 그룹을 통째로 업데이트
    - 5점 이상 시 5점, 1점 이하 시 1점 >> 1점 추가 시 5그룹 = 4그룹+5그룹
    
    - 합집합이 빠르게 되는 자료구조 뭐가 있지? >> linked list (set은 느림)
    - linked 리스트 쓰면 두 linkend 리스트를 연결하는 것을 상수 시간 내에 가능
    - 각 팀별로 병사를 또 점수별로 그룹화. linked list로 관리 (두 집합을 연결할 필요성)


- 풀이

1. 고용
팀 배열에 사람 추가

2. 해고
먼저 사람을 찾아야 함(N) 느림 >> 배열로 관리 불가
- hard delete와 soft delete
  - 직접 병사를 찾아서 삭제하는 hard delete
  - 오래 걸림 >> soft delete
  - 업데이트와 유사한 방식. 직접 찾아서 지우지 않고 버전을 -1 로 변경

3. 특정 병사 점수 변경
  - 병사의 버전을 추가해서 이전 버전의 병사를 무시
  - n번 병사 평판을 2에서 4로 변경한다면, 기존에는 2를 삭제하고 4에 추가
  - but 2는 냅두고 버전 올린 병사를 4에 추가
  - 동일 id 병사 많겠지만 버전만 신경씀

4. 특정 팀 모든 병사 평판 변경 (중요!)
  - 이 조건으로 인해 Linked list 자료구조가 유리
